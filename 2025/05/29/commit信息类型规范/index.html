commit信息类型规范

发表于 2025-05-29 分类于 GIT 阅读次数：

在编写git commit信息时，可以遵循以下规范：

类型 说明
fea 新功能
fix 修复bug
refactor 重构功能，不新增功能，也不修 bug
perf 性能优化
style 改代码风格，不影响功能
test 加测试、改测试
docs 改文档，比如 README
chore 杂项，比如改 .gitignore、构建脚本
ci CI/CD 相关改动
build 改构建系统或依赖
revert 回滚某个提交

例如：

fea(用户模块)：

新增用户接口。

fix(用户模块)：

修改接口报错问题修复。
